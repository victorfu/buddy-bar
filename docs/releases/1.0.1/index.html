BuddyBar Release Notes

BuddyBar 1.0.1

April 6, 2026 · View on GitHub

Added

  • Added a Sparkle-backed in-app updater path with a manual Check for Updates… action in BuddyBar's About settings.
  • Added real usage reporting with Anthropic OAuth rate-limit data, local usage stats, and richer per-session metrics in the Buddy UI.
  • Added lightweight system health monitoring, service-management launch-at-login controls, and self-service hook repair flows to the native settings experience.

Improved

  • Improved BuddyBar's release path so the private source repo can publish signed DMGs, changelog-sourced release notes, and public appcast metadata to the dedicated distribution repo.
  • Improved menu bar and settings polish with pixel-art iconography, clearer usage presentation, and better focus behavior when reopening from notifications.
  • Improved bridge installation and runtime integration so local hooks, desktop notifications, and updater metadata all align on stable paths and public URLs.

Fixed

  • Fixed approval flows that could stay stuck after external resolution or get clobbered by late-arriving bridge events.
  • Fixed popover and notification hand-off behavior so session reopening, terminal jumping, and notification cleanup behave more predictably.
  • Fixed release-notes and appcast generation paths to align with the proven Sparkle flow already exercised in SnapTray.
